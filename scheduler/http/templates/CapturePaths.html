Capture Paths
Multiple capture paths/drives can be added and used and when creating a schedule item you can pick which path/drive to use for the capture. You can also select the path/drive to use when an Auto-Add item adds a new schedule item.

By default when adding new schedule items and Auto-Add items the AutoSelect option will selected. AutoSelect will chose the best path/drive based on the AutoSelect type (see AutoSelect Type below).

Multiple Path Settings
Minimum Free Space (Soft):
This value is used to determine the amount of free space available on a path before a capture is started on that particular path. At least this much free space is required before a capture can be started on a path.

Minimum Free Space (Hard):
The amount of free space at which point a capture will be stopped. The capture is then marked for restart and on restarting a new path is selected based on your AutoSelect options. Hopefully there is another path set up that has enough space left to take the capture.

Delete to Free Space
When selected this will try to deleted the oldest auto delete captures, these are captures that have the auto delete set to true in their options. The oldest will be deleted until the soft free space limit is reached.

AutoSelect Type:
There are two types of AutoSelect, the first picks the drive with the most available free space from your list of paths/drives, this makes sure your capture has the best chance for completing without having to stop the capture and select a new path/drive to complete it.
The Second AutoSelect type picks the first path/drive with enough space (x2 the minimum free space value) and uses that. This might not be the drive with the most space and the capture might have to move on to a new path/drive as it fills it up with capture data.
This is a global setting and applies to all schedules that use the AutoSelect option for their path.

Include Proposed Capture:
When checked takes into account how much drive space would be required to do the proposed capture, it uses the Capture Data Rate (see below) and the capture duration to calculate the amount of space for the proposed capture and then subtracts this form the free space of the path/drive before doing the Minimum Free Space calculation (see Minimum Free Space above). This means that it should not be necessary to split the capture across multiple paths as a path with enough space should be selected.

Capture Data Rate
This is the average data rate in bits per second for the full capture, this includes everything that is captured to the file thus it might be more for some channels with multiple audio tracks and HD channels. It might also be more for Full TS captures, as you can see this is an indication only and you can set this higher than needed to make sure you allow for enough space for the capture.

File Name Patterns
When TV Scheduler Pro runs it uses a pattern match substitution to create the file name to capture to, this pattern match uses the following keys to do the replace:

%y =Year
%m =Month
%d =Day of Month
%h =Hour 24 hour time
%M =Minute
%n =The name of the item
%s =The sub title of the EPG item the schedule was created from
%t =The category of the EPG item the schedule was created from
%c =Channel name
%D =Day of week

You can also specify directory sub paths and if the path does not exist TV Scheduler Pro will automatically create it for you.

You can also add the word 'default' as a pseudo pattern. Any schedule that has the filename pattern set as 'default' will substitute the current first real pattern in the pattern list each time a schedule is created for it. 'default' can be set as the first item in the list so that when adding new schedules/auto-adds 'default' is the first pattern shown. The second pattern will be substituted in this case. It can also be moved down the list if preferred and only applied sometimes. It can also be deleted and/or added back in at a later date.
